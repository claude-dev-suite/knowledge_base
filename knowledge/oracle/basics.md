# Oracle Database Deep Dive

## Oracle Architecture

### Database vs Instance
- **Database**: Physical files (datafiles, control files, redo logs)
- **Instance**: Memory structures (SGA) + background processes
- Multiple instances can mount same database (RAC)

### System Global Area (SGA)

```
SGA Components:
├── Database Buffer Cache (data blocks)
├── Shared Pool
│   ├── Library Cache (parsed SQL)
│   └── Data Dictionary Cache
├── Redo Log Buffer
├── Large Pool (RMAN, parallel query)
├── Java Pool (Java stored procedures)
└── Streams Pool (Oracle Streams)
```

### Background Processes

| Process | Function |
|---------|----------|
| PMON | Process monitor, cleanup |
| SMON | System monitor, recovery |
| DBWn | Database writer (buffer → datafile) |
| LGWR | Log writer (redo buffer → redo log) |
| CKPT | Checkpoint process |
| ARCn | Archiver (redo log → archive) |
| MMON | Manageability monitor |
| MMNL | Memory monitor light |

## Oracle Data Types

### Numeric Types

```sql
-- NUMBER: Most flexible
NUMBER              -- Up to 38 digits
NUMBER(10)          -- Precision 10, scale 0 (integer)
NUMBER(10,2)        -- Precision 10, scale 2 (2 decimals)
NUMBER(10,-2)       -- Rounded to nearest 100

-- Native types (faster)
BINARY_FLOAT        -- IEEE 754 single precision
BINARY_DOUBLE       -- IEEE 754 double precision

-- Integer subtypes
INTEGER             -- NUMBER(38)
SMALLINT           -- NUMBER(38)
```

### Character Types

```sql
-- Fixed length (space padded)
CHAR(10)            -- Max 2000 bytes
NCHAR(10)           -- Unicode, max 1000 chars

-- Variable length
VARCHAR2(100)       -- Max 4000 bytes (32767 in PL/SQL)
NVARCHAR2(100)      -- Unicode, max 2000 chars

-- Large objects
CLOB                -- Character large object (up to 4GB)
NCLOB               -- Unicode CLOB
BLOB                -- Binary large object
BFILE               -- External file reference
```

### Date and Time Types

```sql
-- DATE: date + time (no fractional seconds)
DATE                -- 7 bytes

-- TIMESTAMP: date + time + fractional seconds
TIMESTAMP           -- Default 6 digits fractional
TIMESTAMP(9)        -- 9 digits fractional
TIMESTAMP WITH TIME ZONE
TIMESTAMP WITH LOCAL TIME ZONE

-- INTERVAL: duration
INTERVAL YEAR TO MONTH
INTERVAL DAY TO SECOND
INTERVAL '2-6' YEAR TO MONTH    -- 2 years 6 months
INTERVAL '5 12:30:00' DAY TO SECOND
```

## Sequences

```sql
-- Create sequence
CREATE SEQUENCE order_seq
    START WITH 1000
    INCREMENT BY 1
    MAXVALUE 999999999
    MINVALUE 1
    CYCLE
    CACHE 20
    ORDER;      -- Guarantees order (slower)

-- Use sequence
SELECT order_seq.NEXTVAL FROM dual;
SELECT order_seq.CURRVAL FROM dual;  -- After NEXTVAL in same session

-- In INSERT
INSERT INTO orders (id, customer_id)
VALUES (order_seq.NEXTVAL, 100);

-- Identity column (12c+)
CREATE TABLE orders (
    id NUMBER GENERATED ALWAYS AS IDENTITY,
    customer_id NUMBER
);

-- Or GENERATED BY DEFAULT (allows explicit values)
CREATE TABLE orders (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY
        START WITH 1000 INCREMENT BY 1,
    customer_id NUMBER
);
```

## Partitioning

### Range Partitioning

```sql
CREATE TABLE sales (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (sale_date) (
    PARTITION sales_2023_q1 VALUES LESS THAN (DATE '2023-04-01'),
    PARTITION sales_2023_q2 VALUES LESS THAN (DATE '2023-07-01'),
    PARTITION sales_2023_q3 VALUES LESS THAN (DATE '2023-10-01'),
    PARTITION sales_2023_q4 VALUES LESS THAN (DATE '2024-01-01'),
    PARTITION sales_future VALUES LESS THAN (MAXVALUE)
);
```

### List Partitioning

```sql
CREATE TABLE customers (
    customer_id NUMBER,
    region VARCHAR2(20)
)
PARTITION BY LIST (region) (
    PARTITION customers_east VALUES ('NY', 'NJ', 'CT'),
    PARTITION customers_west VALUES ('CA', 'WA', 'OR'),
    PARTITION customers_other VALUES (DEFAULT)
);
```

### Hash Partitioning

```sql
CREATE TABLE transactions (
    txn_id NUMBER,
    account_id NUMBER
)
PARTITION BY HASH (account_id)
PARTITIONS 8;
```

### Composite Partitioning

```sql
CREATE TABLE orders (
    order_id NUMBER,
    order_date DATE,
    customer_id NUMBER
)
PARTITION BY RANGE (order_date)
SUBPARTITION BY HASH (customer_id)
SUBPARTITIONS 4
(
    PARTITION orders_2023 VALUES LESS THAN (DATE '2024-01-01'),
    PARTITION orders_2024 VALUES LESS THAN (DATE '2025-01-01')
);
```

### Interval Partitioning (11g+)

```sql
-- Auto-creates partitions as needed
CREATE TABLE logs (
    log_id NUMBER,
    log_date DATE
)
PARTITION BY RANGE (log_date)
INTERVAL (NUMTOYMINTERVAL(1, 'MONTH'))
(
    PARTITION logs_initial VALUES LESS THAN (DATE '2023-01-01')
);
```

### Partition Maintenance

```sql
-- Add partition
ALTER TABLE sales ADD PARTITION sales_2024_q1
    VALUES LESS THAN (DATE '2024-04-01');

-- Drop partition
ALTER TABLE sales DROP PARTITION sales_2023_q1;

-- Truncate partition (fast)
ALTER TABLE sales TRUNCATE PARTITION sales_2023_q1;

-- Move partition to different tablespace
ALTER TABLE sales MOVE PARTITION sales_2023_q2
    TABLESPACE new_ts;

-- Split partition
ALTER TABLE sales SPLIT PARTITION sales_future
    AT (DATE '2024-04-01')
    INTO (PARTITION sales_2024_q1, PARTITION sales_future);

-- Merge partitions
ALTER TABLE sales MERGE PARTITIONS sales_2023_q1, sales_2023_q2
    INTO PARTITION sales_2023_h1;

-- Exchange partition (swap with table)
ALTER TABLE sales EXCHANGE PARTITION sales_2023_q1
    WITH TABLE sales_2023_q1_staging;
```

## Flashback Technology

### Flashback Query

```sql
-- Query as of timestamp
SELECT * FROM employees
AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);

-- Query as of SCN
SELECT * FROM employees
AS OF SCN 123456789;

-- Get current SCN
SELECT CURRENT_SCN FROM V$DATABASE;
```

### Flashback Version Query

```sql
-- All versions of rows between two points
SELECT versions_starttime, versions_endtime, versions_operation,
       employee_id, salary
FROM employees
VERSIONS BETWEEN TIMESTAMP
    SYSTIMESTAMP - INTERVAL '2' HOUR
    AND SYSTIMESTAMP
WHERE employee_id = 100;
```

### Flashback Table

```sql
-- Enable row movement
ALTER TABLE employees ENABLE ROW MOVEMENT;

-- Flashback to timestamp
FLASHBACK TABLE employees
TO TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);

-- Flashback to SCN
FLASHBACK TABLE employees TO SCN 123456789;

-- Flashback to before drop
FLASHBACK TABLE dropped_table TO BEFORE DROP;
-- Or with rename
FLASHBACK TABLE dropped_table TO BEFORE DROP
    RENAME TO restored_table;
```

### Flashback Database

```sql
-- Requires FLASHBACK DATABASE privilege and flashback logs
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
FLASHBACK DATABASE TO TIMESTAMP
    TO_TIMESTAMP('2024-01-15 10:00:00', 'YYYY-MM-DD HH24:MI:SS');
ALTER DATABASE OPEN RESETLOGS;
```

## Materialized Views

```sql
-- Create materialized view
CREATE MATERIALIZED VIEW mv_sales_summary
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
ENABLE QUERY REWRITE
AS
SELECT product_id, SUM(quantity) total_qty, SUM(amount) total_amount
FROM sales
GROUP BY product_id;

-- Create materialized view log (for fast refresh)
CREATE MATERIALIZED VIEW LOG ON sales
WITH ROWID, SEQUENCE (product_id, quantity, amount)
INCLUDING NEW VALUES;

-- Manual refresh
BEGIN
    DBMS_MVIEW.REFRESH('mv_sales_summary', 'C');  -- Complete
    DBMS_MVIEW.REFRESH('mv_sales_summary', 'F');  -- Fast
END;
```

## Optimizer Hints

```sql
-- Index hints
SELECT /*+ INDEX(e emp_idx) */ * FROM employees e;
SELECT /*+ NO_INDEX(e emp_idx) */ * FROM employees e;
SELECT /*+ INDEX_FFS(e emp_idx) */ * FROM employees e;  -- Fast full scan

-- Join hints
SELECT /*+ USE_NL(e d) */ *    -- Nested loops
FROM employees e, departments d WHERE e.dept_id = d.id;

SELECT /*+ USE_HASH(e d) */ *  -- Hash join
FROM employees e, departments d WHERE e.dept_id = d.id;

SELECT /*+ USE_MERGE(e d) */ * -- Sort merge
FROM employees e, departments d WHERE e.dept_id = d.id;

-- Parallelism
SELECT /*+ PARALLEL(e, 4) */ * FROM employees e;
SELECT /*+ PARALLEL(4) */ * FROM employees;  -- All tables

-- Other useful hints
SELECT /*+ FIRST_ROWS(10) */ * FROM orders;  -- Optimize for first N
SELECT /*+ ALL_ROWS */ * FROM orders;         -- Optimize for throughput
SELECT /*+ FULL(e) */ * FROM employees e;     -- Full table scan
SELECT /*+ ORDERED */ * FROM t1, t2, t3;      -- Join in FROM order
SELECT /*+ LEADING(t2 t1) */ * FROM t1, t2;   -- Specify join order
```

## PL/SQL Packages

### Package Specification

```sql
CREATE OR REPLACE PACKAGE order_pkg AS
    -- Constants
    c_default_status CONSTANT VARCHAR2(20) := 'PENDING';

    -- Types
    TYPE order_rec IS RECORD (
        order_id NUMBER,
        customer_id NUMBER,
        total NUMBER
    );
    TYPE order_tab IS TABLE OF order_rec;

    -- Procedures
    PROCEDURE create_order(
        p_customer_id IN NUMBER,
        p_order_id OUT NUMBER
    );

    -- Functions
    FUNCTION get_order_total(p_order_id IN NUMBER) RETURN NUMBER;

    -- Cursor
    CURSOR c_pending_orders IS
        SELECT * FROM orders WHERE status = 'PENDING';
END order_pkg;
/
```

### Package Body

```sql
CREATE OR REPLACE PACKAGE BODY order_pkg AS
    -- Private variables
    g_last_order_id NUMBER;

    -- Private procedure
    PROCEDURE log_order(p_order_id NUMBER) IS
    BEGIN
        INSERT INTO order_log VALUES (p_order_id, SYSDATE);
    END log_order;

    -- Public procedure implementation
    PROCEDURE create_order(
        p_customer_id IN NUMBER,
        p_order_id OUT NUMBER
    ) IS
    BEGIN
        INSERT INTO orders (customer_id, status)
        VALUES (p_customer_id, c_default_status)
        RETURNING order_id INTO p_order_id;

        g_last_order_id := p_order_id;
        log_order(p_order_id);
    END create_order;

    -- Public function implementation
    FUNCTION get_order_total(p_order_id IN NUMBER) RETURN NUMBER IS
        v_total NUMBER;
    BEGIN
        SELECT SUM(quantity * price) INTO v_total
        FROM order_items
        WHERE order_id = p_order_id;
        RETURN NVL(v_total, 0);
    END get_order_total;

-- Package initialization
BEGIN
    g_last_order_id := 0;
END order_pkg;
/
```

## Autonomous Transactions

```sql
CREATE OR REPLACE PROCEDURE log_error(p_message VARCHAR2) AS
    PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
    INSERT INTO error_log (message, created_at)
    VALUES (p_message, SYSDATE);
    COMMIT;  -- Commits only this autonomous transaction
END;
/

-- Usage: Error logged even if main transaction rolls back
BEGIN
    -- Some operation
    UPDATE accounts SET balance = balance - 100;

    -- Log (commits independently)
    log_error('Debit processed');

    -- Main transaction fails
    RAISE_APPLICATION_ERROR(-20001, 'Error!');
EXCEPTION
    WHEN OTHERS THEN
        log_error('Transaction failed: ' || SQLERRM);
        ROLLBACK;  -- Main rollback, but log entries persist
END;
```

## Database Links

```sql
-- Create database link
CREATE DATABASE LINK remote_db
CONNECT TO remote_user IDENTIFIED BY password
USING 'tns_alias';

-- Use database link
SELECT * FROM employees@remote_db;

-- Insert across link
INSERT INTO local_employees
SELECT * FROM employees@remote_db;

-- Synonym for remote object
CREATE SYNONYM remote_employees
FOR employees@remote_db;
```
