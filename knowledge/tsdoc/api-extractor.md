# API Extractor

> Source: https://api-extractor.com/

## Overview

API Extractor is a tool that:
1. Generates API documentation from TSDoc comments
2. Creates .d.ts rollup files
3. Tracks API changes and reviews
4. Enforces documentation standards

## Installation

```bash
npm install --save-dev @microsoft/api-extractor @microsoft/api-documenter
```

## Configuration

### api-extractor.json

```json
{
  "$schema": "https://developer.microsoft.com/json-schemas/api-extractor/v7/api-extractor.schema.json",
  "mainEntryPointFilePath": "<projectFolder>/dist/index.d.ts",
  "bundledPackages": [],

  "compiler": {
    "tsconfigFilePath": "<projectFolder>/tsconfig.json"
  },

  "apiReport": {
    "enabled": true,
    "reportFileName": "<unscopedPackageName>.api.md",
    "reportFolder": "<projectFolder>/etc/",
    "reportTempFolder": "<projectFolder>/temp/"
  },

  "docModel": {
    "enabled": true,
    "apiJsonFilePath": "<projectFolder>/temp/<unscopedPackageName>.api.json"
  },

  "dtsRollup": {
    "enabled": true,
    "untrimmedFilePath": "<projectFolder>/dist/<unscopedPackageName>.d.ts",
    "publicTrimmedFilePath": "<projectFolder>/dist/<unscopedPackageName>-public.d.ts"
  },

  "tsdocMetadata": {
    "enabled": true,
    "tsdocMetadataFilePath": "<projectFolder>/dist/tsdoc-metadata.json"
  },

  "messages": {
    "compilerMessageReporting": {
      "default": { "logLevel": "warning" }
    },
    "extractorMessageReporting": {
      "default": { "logLevel": "warning" },
      "ae-missing-release-tag": { "logLevel": "none" }
    },
    "tsdocMessageReporting": {
      "default": { "logLevel": "warning" }
    }
  }
}
```

## Running API Extractor

```bash
# Initialize configuration
npx api-extractor init

# Run extraction
npx api-extractor run

# Run in local mode (doesn't fail on warnings)
npx api-extractor run --local

# Verbose output
npx api-extractor run --verbose
```

## API Report

The API report tracks your public API surface.

### Example: etc/my-package.api.md

```markdown
## API Report File for "my-package"

> Do not edit this file. It is a report generated by API Extractor.

### @public

export function add(a: number, b: number): number;

export class Calculator {
  constructor(initial?: number);
  add(value: number): this;
  readonly result: number;
}

export interface Config {
  debug?: boolean;
  timeout?: number;
}

export type EventHandler = (event: Event) => void;
```

## Release Tags

API Extractor uses TSDoc release tags:

```typescript
/**
 * @public - Stable, documented API
 */
export function stableApi(): void;

/**
 * @beta - Preview, may change
 */
export function previewApi(): void;

/**
 * @alpha - Experimental
 */
export function experimentalApi(): void;

/**
 * @internal - Excluded from public docs
 */
export function internalHelper(): void;
```

## Trimmed Declarations

API Extractor can generate trimmed .d.ts files:

```
dist/
├── my-package.d.ts           # Full declarations
├── my-package-public.d.ts    # Only @public
└── my-package-beta.d.ts      # @public + @beta
```

Configuration:

```json
{
  "dtsRollup": {
    "enabled": true,
    "untrimmedFilePath": "./dist/my-package.d.ts",
    "publicTrimmedFilePath": "./dist/my-package-public.d.ts",
    "betaTrimmedFilePath": "./dist/my-package-beta.d.ts"
  }
}
```

## Message Severity Levels

```json
{
  "messages": {
    "extractorMessageReporting": {
      "ae-missing-release-tag": {
        "logLevel": "warning",
        "addToApiReportFile": true
      },
      "ae-unresolved-link": {
        "logLevel": "error"
      },
      "ae-forgotten-export": {
        "logLevel": "warning"
      }
    }
  }
}
```

### Common Message IDs

| ID | Description |
|----|-------------|
| `ae-missing-release-tag` | Export lacks @public/@beta/@alpha |
| `ae-forgotten-export` | Type used but not exported |
| `ae-unresolved-link` | TSDoc @link target not found |
| `ae-incompatible-release-tags` | Child more visible than parent |
| `ae-undocumented` | Missing documentation |

## Complete Workflow

```typescript
// src/index.ts

/**
 * A utility library for math operations.
 *
 * @packageDocumentation
 */

/**
 * Adds two numbers.
 *
 * @example
 * ```ts
 * add(1, 2); // Returns 3
 * ```
 *
 * @param a - First number
 * @param b - Second number
 * @returns The sum of a and b
 *
 * @public
 */
export function add(a: number, b: number): number {
  return a + b;
}

/**
 * Calculator class for chained operations.
 *
 * @example
 * ```ts
 * const calc = new Calculator(10);
 * calc.add(5).multiply(2).result; // 30
 * ```
 *
 * @public
 */
export class Calculator {
  private _value: number;

  /**
   * Creates a new Calculator.
   * @param initial - Initial value
   */
  constructor(initial: number = 0) {
    this._value = initial;
  }

  /**
   * Current result.
   * @readonly
   */
  get result(): number {
    return this._value;
  }

  /**
   * Adds to current value.
   * @param n - Number to add
   */
  add(n: number): this {
    this._value += n;
    return this;
  }

  /**
   * @internal
   */
  _debugValue(): void {
    console.log(this._value);
  }
}
```

## CI Integration

```yaml
# GitHub Actions
- name: Build
  run: npm run build

- name: API Extractor
  run: npx api-extractor run

- name: Check API report
  run: |
    if git diff --exit-code etc/; then
      echo "API report is up to date"
    else
      echo "API report needs update"
      exit 1
    fi
```

## Best Practices

1. **Commit API reports** - Track API changes in version control
2. **Review API changes** - API report diffs show breaking changes
3. **Use release tags** - Mark stability of every export
4. **Document public API** - Every @public item should have docs
5. **Run in CI** - Catch undocumented exports automatically
